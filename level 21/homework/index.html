თქვენი სიტყვებით ახსენით რა არის და როგორ მუშაობს priority'ები

1. Inline Styles: Highest priority. Styles applied directly within an HTML element using the style attribute override any internal or external styles.

2. IDs: High priority. Styles defined using ID selectors (e.g., #myId) will override class and element styles, but not inline styles.

3. Classes: Medium priority. Styles applied using class selectors (e.g., .myClass) can override element styles but are overridden by inline styles and ID styles.

4. Elements: Lowest priority. Styles applied using element selectors (e.g., div, p) can be overridden by any of the above styles.

თქვენი სიტყვებით ახსენით რა არის და როგორ მუშაობს style inheritance

1. Inherited Properties: Some CSS properties, such as color, font-family, and line-height, are inherited by child elements from their parent elements. For example, if a parent "div" has a specific text color, all text inside that "div" will inherit that color unless specified otherwise.

2. Non-Inherited Properties: Other properties, like margin, padding, and border, are not inherited. They apply only to the element they are defined on, requiring child elements to have their own styles if they need different margins or paddings.

3. Cascading Effect: Inheritance works alongside the CSS cascade, where styles can be applied at various levels (inline, internal, external). Inherited styles can be overridden by more specific rules.

4. Using the inherit Keyword: You can explicitly make a property inherit by using the inherit value. For example, setting font-size: inherit; on a child element will make it inherit the font size from its parent.
